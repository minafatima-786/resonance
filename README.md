## 🎯 Project Overview & Problem Statement

### The Real Problem
In high-pressure academic and professional settings, people meticulously track metrics like fitness, steps, or sleep hours, but rarely examine how these physical behaviors structurally impact their psychological states. This creates a data blindness where individuals cannot accurately pinpoint what variables are actively degrading their emotional stability, leading to unpredictable spikes in stress and eventual burnout.

### The Solution: Resonance
Resonance is an analytical emotional intelligence co-pilot designed to act as an early-intercept behavioral tracking system. By combining quantitative multiple linear regression analysis with advanced generative psychological modeling, Resonance isolates exactly how subtle changes in sleep patterns and stress levels manifest as mood variance. It empowers users to understand their structural emotional thresholds and deploy targeted, non-reactive behavioral responses before crisis points occur.

## 📐 Mathematical Framework & Regression Engine

Resonance doesn't just display data—it calculates behavioral impact weights using an in-memory **Multiple Linear Regression (MLR)** algorithm via the `/api/analyze` endpoint. 

The application models your mood state using the following mathematical formulation:

\[\text{Mood} = \beta_0 + \beta_1(\text{Sleep}) + \beta_2(\text{Stress}) + \epsilon\]

Where:
*   β₀ is the intercept baseline state.
*   β₁ represents the **Sleep Coefficient Impact Weight** (multiplied by 10 on the dashboard dashboard).
*   β₂ represents the **Stress Coefficient Impact Weight** (multiplied by 10 on the dashboard dashboard).
*   ε represents residual emotional variance.

By evaluating the past 14 logs dynamically, the engine isolates which environmental factor has the highest statistical covariance with emotional degradation, allowing the AI to customize its prescriptive feedback based on your actual data trends.

## 🗄️ Database Schema Layer (PostgreSQL DDL)

The data layer is hosted securely on a relational PostgreSQL database with strict constraints to ensure zero data corruption:

```sql
CREATE TABLE daily_logs (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL,
  date DATE NOT NULL,
  sleep NUMERIC(3,1) CHECK (sleep >= 0 AND sleep <= 12),
  stress NUMERIC(3,1) CHECK (stress >= 0 AND stress <= 10),
  mood NUMERIC(3,1) CHECK (mood >= 0 AND mood <= 10),
  trigger_category TEXT CHECK (trigger_category IN ('Work', 'Family', 'Health', 'Finance', 'None')),
  is_baseline BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Enable Security Layers
ALTER TABLE daily_logs ENABLE ROW LEVEL SECURITY;

-- Establish Explicit User Isolation Isolation
CREATE POLICY "Users can manage own logs" ON daily_logs FOR ALL USING (auth.uid() = user_id);
```

## 🤖 The AI Feature & Core System Prompt

### Behavioral Reframing Engine
When a user interacts with **The Red Button** interface, the application packages their current behavioral coefficients (calculated sleep impact weight, stress impact weight, and worst historical trigger category) alongside their raw, unedited text entry. This payload is dispatched to a structured Cohere AI agent using the specialized `command-r-08-2024` model. The agent acts as a behavioral psychologist to convert reactive emotional output into a structured, actionable output matrix.

### Exact System Instructions & Prompt
The agent is explicitly governed by the following system instructions to guarantee absolute analytical precision and valid object parsing:

```text
You are a data-driven behavioral psychologist. Regression analysis shows Sleep weight = {coefficients.sleep}, Stress weight = {coefficients.stress}. The user's worst trigger is {worstTrigger}. If user sent text: "{userText}" and trigger is "{triggerType}". Generate a JSON response with exactly these keys: 'prescription' (1-sentence morning action), 'grounding' (15-second breathing mantra), 'reframed' (rewrite their angry text into a diplomatic message), 'insight' (1-sentence weekly insight). Output ONLY valid JSON. No markdown, no extra text.
```

## 🚀 How to Run the Project Locally

1. **Clone the repository workspace:**
   ```bash
   git clone https://github.com
   cd resonance
   ```

2. **Install all required dependencies and node modules:**
   ```bash
   npm install @supabase/supabase-js cohere-ai recharts regression @fontsource/playfair-display @fontsource/inter
   ```

3. **Configure your local system environment variables (`.env.local`):**
   ```env
   NEXT_PUBLIC_SUPABASE_URL=your_supabase_project_url
   NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_public_anon_key
   COHERE_API_KEY=your_cohere_ai_api_key
   ```

4. **Boot up the local optimization development network server:**
   ```bash
   npm run dev
   ```

5. Open your local network connection space at [http://localhost:3000](http://localhost:3000) to review the app.
